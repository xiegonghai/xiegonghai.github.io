Hexo博客搭建

发表于 2016-03-01   |   分类于 博客   |  

前言

近来想着搭建一个基于github的博客站点,之前在csdn的博客频繁性恶心弹窗广告，后面虽说装adblock自己是看不到了，但别人浏览还是会有的，大概了解了下，比较好的博客模板有octopress,wordpress,hexo几种，octopress还不错，开始搭建之后，由于版本更新匹配的原因，总是会有小错误，加之对ruby不熟，于是切换到hexo来搭建

搭建教程

主要参考http://www.mashiro.io/2015/09/hexo-guide-2/，写的很详细，就不重复 造轮子了，仔细看完ok了

安装Git

去 Git 官网下载并安装即可.

安装Node.Js

去NodeJs 官网下载并安装即可.
注意: 尽量使用默认路径, 以保证环境变量的自动配置.

安装Hexo

使用npm命令即可直接安装 (可以直接在屏幕任意地方直接右键: Git Bash Here)

静静等待安装完毕即可.

创建Hexo目录

目录可以再随意地方创建 (注:尽量全英文路径 避免不必要的问题)
EG: C:\Users\Xuning\Documents\Hexo (这是我的本地路径)
现在进入Hexo目录内在里面右键选择: Git Bash Here

Hexo 会自动生成建站所需要的文件

npm 会自动安装依赖环境

运行

Hexo 运行方法很简单只需要一条命令

客户端

与服务器建立连接，返回一个套接字描述符
主要调用socket,connect这两个函数

/ $end open_clientfd /

成功返回0,出错返回-1
告诉内核将my_addr中的服务器套接字地址与套接字描述符sockfd联系起来,参数addrlen＝sizeof(sockaddr_in)

listen

将sockfd从一个主动套接字转化为一个监听套接字-可以接受来自客户端的连接请求

accept函数

成功返回非负已连接描述符，出错返回-1
等待来自客户端的连接请求到达侦听描述符listenfd，然后在addr中填写客户端的套接字地址，返回一个已连接描述符

总结

为何要有监听描述符合已连接描述符区别

主要为了建立并发服务器,它能够同时处理许多客户端连接,例如每一个连接请求到达
监听描述符时,可以派生一个新的进程通过已连接描述符与客户端通信

客户端主要流程

socket->connect
调用socket创建套接字描述符,调用connect和服务器ip+iport进行连接,
阻塞在connect

服务器主要流程

socket->bind->listen->(多个accept)
调用socket创建套接字描述符,调用bind将ip+port与套接字描述符绑定调用listen告诉内核socket是被服务器而非客户端使用,将一个主要套接字转换为监听套接字
